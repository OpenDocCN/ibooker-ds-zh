# 第二部分. 管理者：培养团队

如果你已经证明了管理技术项目的能力和美德，你可能已经准备好承担作为团队数据科学家管理团队成员或管理更广泛的技术项目范围的责任。作为一个人员管理者或团队数据科学家，你的组织依赖你赋予你的数据科学团队在多个项目中尽可能高效的能力。你的团队成员依赖你赋予他们做到职业生涯中最佳工作的能力。你如何才能满足并超越如此高的期望？

范围更广意味着责任更大。你的责任不仅限于管理人员。管理团队的*目的*是监督技术能力的建设。*方法*是通过招聘、指导和绩效管理数据科学家来建立一个有能力的专业团队。团队将把他们的经理和员工数据科学家视为领域专业知识和技术洞察力的榜样。我们在第四章中讨论了目的和许多方法。

要通过数据科学产生积极影响，你可以在团队中培养第三章中描述的数据科学美德，包括道德、严谨和态度，以及练习额外的人员管理美德，这些我们在第五章中讨论。

成功的数据科学管理者对数据科学领域充满热情。他们承担更多责任，以产生更大的影响。人员管理只是实现更大影响的手段。

埃里克·科尔森（Eric Colson），Stitch Fix 的前首席算法官，以及 Netflix 和 Yahoo 的前数据科学高管，曾写道：“根据我的经验，许多最好的数据科学领导者对管理持矛盾态度。他们热爱脚本编写、编码、分析、讲故事、构建框架等等，并且不愿意放弃这些。但最终，他们更爱影响。” [1]。

管理路径并不是数据从业者晋升的唯一途径。许多数据科学组织提供技术路径，随着职能责任的增长，可以领导项目和影响业务决策。如果你选择技术路径的职业，你可以参考第 4.1 节和第 4.3 节的技术人员级别里程碑，以及第 6.1 节和第 6.3 节的主管级别里程碑。

如果你立志最终成为一名数据高管，那么成为管理者的道路，我们将在第四章和第五章中详细描述，将使你有机会练习成功高管所必需的能力和美德。让我们开始吧！
