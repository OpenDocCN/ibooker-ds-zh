# 第三部分：总监：治理一个职能

通过你在项目管理中的卓越诚信和在领导一支高效的数据科学家团队中的细微方法，你已准备好承担更大的责任。你可以领导一个由多个团队组成的 DS 职能，管理经理，并解决你直接见证不到的挑战。你也可以作为主要数据科学家在个人贡献者轨道上，影响更广泛的合作伙伴，并领导更复杂的努力。

在 DS 职能层面进行领导需要与领导项目或个人团队所需的技能不同。你负责在更长的时间范围内产生更显著的影响，通常跨越多个季度。这一角色的区别在于对具有更长时间跨度的项目所需的专注度和优先级排序的清晰度。

专注度和优先级排序的清晰度来自于对你公司商业模式的深刻理解。你可以利用这种理解来制定路线图以实现战略商业目标，同时避免管理和技术陷阱。

一个有效的路线图将最终目标分解为团队领导者可以执行的逐步策略。执行这些路线图需要敏锐地感知组织中的新兴问题，包括技术和与人员相关的问题。只有当里程碑成功完成，才能实现最终的商业目标。

高效执行通常需要针对特定公司倡议的特定团队组织。随着时间的推移，你可以通过管理其人员、流程和平台来构建和迭代重组团队，以构建职能。为了长期执行奠定坚实的基础，你还需要负责为你的人才管道创建稳健的招聘流程和为团队成员提供清晰的职业道路。

这些策略的成功需要你在公司职能层面领导 DS 团队。在第六章和第七章中，我们将深入探讨你通过技术、执行、专业知识、道德、严谨和态度的讨论所能展现的能力和美德。
