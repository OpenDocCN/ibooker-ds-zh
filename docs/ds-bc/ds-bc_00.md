# 前置内容

## 前言

另一位有希望的候选人未能通过数据科学面试，我开始思考原因。那一年是 2018 年，我正努力扩大我创业公司的数据科学团队。我面试了数十位看似合格的候选人，但最终都拒绝了他们。最新的被拒绝申请者是来自一所顶尖学校的经济学博士。最近，这位申请者在完成了一个为期 10 周的速成班后转入了数据科学领域。我要求这位申请者讨论一个与我们公司非常相关的分析问题。他们立即提到了一个与情况不符的流行算法。当我试图辩论该算法的不兼容性时，候选人陷入了困境。他们不知道该算法实际上是如何工作的，或者应该在什么情况下使用它。这些细节在速成班中并没有教给他们。

在被拒绝的候选人离开后，我开始反思自己的数据科学教育。它与现在有多么不同！回到 2006 年，数据科学还不是一种热门的职业选择，数据科学速成班也尚未存在。那些日子里，我是一名贫穷的研究生，在昂贵的旧金山努力支付房租。我的研究生研究要求我分析数百万个与疾病相关的遗传联系。我意识到我的技能可以转移到其他分析领域，因此我的数据科学咨询公司就这样诞生了。

在我的研究生导师不知情的情况下，我开始向湾区的一些随机公司寻求数据分析工作。这份自由职业的工作帮助我支付了账单，因此我无法对所承担的数据驱动任务过于挑剔。因此，我会报名参加各种数据科学任务，从简单的统计分析到复杂的前瞻性建模。有时我会发现自己被一个看似无法解决的问题所困扰，但最终我会坚持下去。我的挣扎教会了我各种分析技术的细微差别以及如何将它们最佳地结合起来以获得优雅的解决方案。更重要的是，我学会了常见技术可能失败的原因以及如何克服这些失败点以产生有影响力的结果。随着我的技能集的增长，我的数据科学职业生涯开始蓬勃发展。最终，我成为了该领域的领导者。

我是否能在 10 周的速成班通过死记硬背达到同样的成功水平？可能不会。许多速成班更重视独立算法的学习，而不是更连贯的问题解决技能。此外，对算法优势的炒作往往强调其弱点。因此，学生有时在现实世界的环境中处理数据科学时准备不足。这个见解激发了我写这本书的灵感。

我决定通过向您，我的读者，展示一系列越来越具有挑战性的分析问题来复制我自己的数据科学教育。此外，我还选择了为您提供处理这些问题的工具和技术。我的目标是全面帮助您培养分析问题解决能力。这样，当您面试那个初级数据科学职位时，您更有可能获得这份工作。

## 关于本书

开放式问题解决能力对于数据科学职业至关重要。不幸的是，这些能力不能仅仅通过阅读获得。要成为一个问题解决者，你必须坚持不懈地解决难题。本着这个想法，我将我的书结构围绕案例研究来构建：基于现实世界情况的开放式问题。案例研究范围从在线广告分析到利用新闻数据追踪疾病爆发。完成这些案例研究后，你将非常适合开始数据科学职业。

### 应该阅读这本书的人

本书的目标读者是一个受过教育的初学者，他/她对转向数据科学职业感兴趣。当我想象一个典型的读者时，我想象的是一个正在学习经济学的四年级大学生，他/她希望探索更广泛的分析机会，或者一个已经毕业的化学专业学生，他/她在寻找一个更以数据为中心的职业道路。或者，也许读者是一个成功的具有非常有限数学背景的前端网络开发者，他/她想要尝试数据科学。我的所有潜在读者都从未上过数据科学课程，这使得他们在多样化的数据分析方面缺乏经验。本书的目的是消除这种技能缺陷。

我的读者需要了解 Python 编程的裸骨基础知识。自学初学者水平的 Python 应该足以探索书中的练习。你的数学知识预期不会超出基本的高中三角学。

### 本书是如何组织的

本书包含五个难度递增的案例研究。每个案例研究都以一个详细的问题陈述开始，你需要解决它。问题陈述之后是两到五个部分，介绍解决该问题所需的数据科学技能。这些技能部分涵盖了基本库，以及数学和算法技术。每个最终的案例研究部分描述了问题的解决方案。

案例研究 1 涉及基本概率理论：

+   第一部分讨论了如何使用简单的 Python 计算概率。

+   第二部分介绍了概率分布的概念。它还介绍了 Matplotlib 可视化库，该库可用于可视化分布。

+   第三部分讨论了如何使用随机模拟来估计概率。介绍了 NumPy 数值计算库，以促进高效模拟执行。

+   第四部分包含案例研究解决方案。

案例研究 2 将概率扩展到统计学：

+   第五部分介绍了简单的统计中心性和离散度度量。它还介绍了 SciPy 科学计算库，其中包含一个有用的统计模块。

+   第六部分深入探讨了中心极限定理，该定理可用于进行统计预测。

+   第七部分讨论了各种统计推断技术，这些技术可以用来区分有趣的数据模式与随机噪声。此外，本节还说明了错误推断使用的危险以及如何最好地避免这些危险。

+   第八部分介绍了 Pandas 库，该库可用于在统计分析之前预处理表格数据。

+   第九部分包含案例研究解决方案。

案例研究 3 侧重于地理数据的无监督聚类：

+   第十部分说明了如何使用中心性度量将数据聚类成组。同时介绍了 scikit-learn 库以促进高效的聚类。

+   第十一部分侧重于地理数据提取和可视化。使用 GeoNamesCache 库进行文本提取，而使用 Cartopy 地图绘图库实现可视化。

+   第十二部分包含案例研究解决方案。

案例研究 4 侧重于使用大规模数值计算进行自然语言处理：

+   第十三部分说明了如何使用矩阵乘法高效地计算文本之间的相似性。NumPy 内置的矩阵优化被广泛用于此目的。

+   第十四部分展示了如何利用降维技术进行更有效的矩阵分析。数学理论与 scikit-learn 的降维方法相结合进行讨论。

+   第十五部分将自然语言处理技术应用于一个非常大的文本数据集。本节讨论了如何最佳地探索和聚类这些文本数据。

+   第十六部分展示了如何使用 Beautiful Soup HTML 解析库从在线数据中提取文本。

+   第十七部分包含案例研究解决方案。

案例研究 5 通过讨论网络理论和监督机器学习来结束本书：

+   第十八部分介绍了与 NetworkX 图分析库相结合的基本网络理论。

+   第十九部分展示了如何利用网络流在网络数据中找到聚类。概率模拟和矩阵乘法被用于实现有效的聚类。

+   第二十部分介绍了一种基于网络理论的简单监督机器学习算法。还使用 scikit-learn 展示了常见的机器学习评估技术。

+   第二十一部分讨论了依赖于内存高效线性分类器的额外机器学习技术。

+   第二十二部分深入探讨了之前引入的监督学习方法的缺陷。随后使用非线性决策树分类器来规避这些缺陷。

+   第二十三部分包含案例研究解决方案。

书中的每一部分都是基于前一部分介绍过的算法和库构建的。因此，我们鼓励您从头到尾阅读这本书，以最大限度地减少混淆。但如果您已经熟悉书中的一部分内容，您可以自由地跳过这部分熟悉的内容。最后，我强烈建议您在阅读解决方案之前，自己尝试解决每个案例研究问题。独立尝试解决每个问题将最大限度地提高这本书的价值。

### 关于代码

本书包含许多源代码示例，既有编号列表中的，也有与普通文本内联的。在这两种情况下，源代码都使用`固定宽度字体`如这样来与普通文本区分。列表中的源代码以模块化块的形式组织，每个模块化代码块之前都有书面解释。这种代码展示风格非常适合在 Jupyter 笔记本中显示，因为笔记本将功能代码示例与书面解释结合起来。因此，每个案例研究的源代码都可以在 Jupyter 笔记本中下载，网址为 [www.manning.com/books/data-science-bookcamp](http://www.manning.com/books/data-science-bookcamp)。这些笔记本将代码列表与书籍中的总结性解释结合起来。按照常规笔记本风格，不同的笔记本单元之间存在相互依赖关系。因此，建议您按照笔记本中出现的顺序运行代码示例：否则，您可能会遇到由依赖关系驱动的错误。

## 关于封面插图

《数据科学书营》封面上的插图被标注为“特罗尔的居民”，或称特罗尔地区的居民。这幅插图取自雅克·格拉塞·德·圣索沃尔（1757–1810）的作品集，名为《不同国家的服饰》，该作品集于 1797 年在法国出版。每一幅插图都是手工精心绘制和着色的。格拉塞·德·圣索沃尔作品集中的丰富多样性生动地提醒我们，200 年前世界的城镇和地区在文化上有多么不同。人们彼此孤立，说着不同的方言和语言。在街道或乡村，仅凭他们的服饰就能轻易识别他们居住的地方以及他们的职业或社会地位。

自那以后，我们的着装方式已经改变，当时如此丰富的地区多样性已经消失。现在很难区分不同大陆、不同城镇、地区或国家的人们。也许我们用文化多样性换取了更加多样化的个人生活——当然，是为了更加多样化且节奏更快的技术生活。

在难以区分一本计算机书与另一本计算机书的今天，曼宁通过基于两百年前丰富多样的地区生活，并由格拉塞·德·圣索沃尔的画作使之重现的书封面，来庆祝计算机行业的创新精神和主动性。
